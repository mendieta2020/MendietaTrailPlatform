# ADR-NNN — Short Decision Title

- **Status:** proposed | accepted | rejected | deprecated | superseded
- **Outcome:** (optional — e.g. `deferred`, `partial`, `ratified`) — use only when `status` alone is ambiguous
- **Date:** YYYY-MM-DD
- **Decider:** (name / role)
- **Technical author:** (agent or human)
- **Roadmap phase at decision:** P0 | P1 | P2 | P3 | ...
- **Supersedes:** ADR-XXX (optional — if this ADR fully replaces a previous one)
- **Superseded-by:** ADR-XXX (optional — fill in later if another ADR replaces this one)
- **Partially-superseded-by:** ADR-XXX (optional — if a later ADR only replaces part of this decision)
- **Amended-by:** ADR-XXX, ADR-YYY (optional — accumulated list of ADRs that modify clauses without fully replacing)
- **Related skills:** (optional — list of `.claude/skills/` affected or consulted)

---

## Context

What is the situation that forced this decision? Include only facts visible in the repo or verifiable evidence. Do not invent history.

---

## Decision

One paragraph, active voice. State the decision as a commitment, not a suggestion. Include the concrete choice (e.g., "we adopt X", "we defer Y", "we retire Z").

---

## Alternatives Considered

For each alternative, one bullet:

- **Alternative A** — why it was rejected (one sentence).
- **Alternative B** — why it was rejected (one sentence).

Skip this section only if no real alternative existed (rare).

---

## Consequences

### Positive
- What this decision enables.

### Negative / Trade-offs
- What this decision costs or forecloses.

### Neutral / Follow-ups
- What remains open, what needs to happen next, what is out of scope.

---

## Triggers for Re-evaluation

Measurable conditions (NOT dates) that would force revisiting this decision. At least one. Example:

1. Condition X reaches N occurrences.
2. External tool Y transitions from preview to GA.
3. Phase transition from P2 to P3.

If no triggers exist, the decision is permanent — state so explicitly.

---

## Implementation Plan (optional)

If the decision requires code or config changes, list them here. Link to PRs when available. If the ADR is a "documentation-only" decision, omit this section.

---

## Rollback

If this decision reaches production and needs to be reverted, the safest path is: (one sentence). If rollback is not applicable (e.g., a decision not to adopt something), state so.

---

## References

- Files touched or referenced (with paths)
- Related ADRs
- External docs / vendor announcements / commit SHAs
